# Frontend Styling Rules

**Do NOT:**
- Add new color variables or hardcoded colors
- Use bright/saturated colors
- Add decorative elements or excessive styling
- Override existing design tokens

**Design system:** Editorial/financial aesthetic, dark mode primary, warm amber accent used sparingly. Use CSS variables and existing typography/component classes only.

**Full reference:** `docs/claude/frontend-patterns.md` — color palette, typography classes, component patterns, dual-audience messaging system.
